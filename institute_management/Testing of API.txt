Complete API Testing Guide for Institute Management System
1. Environment Setup
Base URL
text
http://localhost:8000/api
Postman Environment Variables
Create these variables in Postman:

base_url: http://localhost:8000/api

access_token: (to be filled after login)

refresh_token: (to be filled after login)

institute_id: (to be filled after institute creation)

student_id: (to be filled after student creation)

teacher_id: (to be filled after teacher creation)

course_id: (to be filled after course creation)

exam_id: (to be filled after exam creation)

result_id: (to be filled after result creation)

2. Authentication APIs
2.1 Register Institute Admin
Endpoint: POST {{base_url}}/auth/register/

Headers:

text
Content-Type: application/json
Body:

json
{
    "username": "institute_admin1",
    "email": "admin1@institute.com",
    "password": "Institute@123",
    "first_name": "John",
    "last_name": "Doe",
    "role": "institute_admin",
    "phone": "1234567890",
    "institute_name": "Oxford Institute"
}
Expected Response (201 Created):

json
{
    "message": "Institute registration successful. Awaiting super admin approval.",
    "user": {
        "id": 2,
        "username": "institute_admin1",
        "email": "admin1@institute.com",
        "first_name": "John",
        "last_name": "Doe",
        "role": "institute_admin",
        "phone": "1234567890",
        "is_approved": false,
        "institute": 1,
        "institute_name": "Oxford Institute"
    }
}
